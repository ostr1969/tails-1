The query_string syntax lets you write powerful search expressions in one line. Use this when you want advanced search logic in a single input box.

Boolean Logic

Combine terms using: AND, OR, NOT, and parentheses.

Examples:

deep learning AND optimization
(nlp OR "natural language") AND transformer

Field Search + Wildcards

Search specific fields. Use wildcards on field names.

Examples:

title:optimization
meta.*:collocation

Fuzziness (Typo Tolerance)

Add ~ to allow misspellings.

Examples:

optimzation~
collocaton~2

Higher numbers allow more edits but can reduce precision.

Word Distance (Proximity Search)

Control how close words must be inside quotes.

Examples:

"nonlinear programming"~5

Smaller numbers = stricter phrase matching.

Range Queries

Filter numeric or date fields with ranges.

Examples:

year:[2018 TO 2024]
date:{2023-01-01 TO 2024-01-01}

[ ] = inclusive, { } = exclusive

_exists_ (Field Presence)

Match documents where a field exists.

Examples:

_exists_:abstract
_exists_:author.name

Boosting (Ranking Control)

Boost important terms using ^.

Examples:

optimization^3 collocation
title:collocation^5 OR body:collocation

Higher boost = more influence on ranking.

Common Pitfalls

  • Special characters must be escaped: + - = && || > < ! ( ) { } [ ] ^ " ~ * ? : \ /
  • Query string is strict: invalid syntax will fail the whole query.
  • Avoid leading wildcards like *term on large indexes (slow).

Example Power Query

(title:optimization^3 OR body:optimization) AND "nonlinear programming"~4 AND _exists_:pdf_url AND year:[2020 TO 2025]
